# Agent notes

Project-specific conventions for `@antfu/design` (the workbench-global rules
still apply).

## Components & stories

- Each component lives in `packages/design/components/<Category>/` and is prefixed
  by its category (`DisplayBadge`, `OverlayModal`, …), with a co-located
  `*.stories.ts`. Keep styling token-driven (no hard-coded colors) and reka-ui for
  overlay behavior.
- Never inline `<svg>` markup for icons. Always use a UnoCSS icon class (`i-ph:*`,
  the Phosphor set already installed via `presetIcons`) on a `<span class="i-ph:..." aria-hidden="true" />`.
  Exception: purely data-driven/generative vector graphics (e.g. `DisplayDonut`'s
  progress ring) aren't icons and stay inline `<svg>`.

## Keep the Storybook Overview up to date

- The single **Overview** page (`packages/design/components/Overview.mdx`) lists
  every component with a linked title + a `<Canvas>` per variation. It is
  **generated**, not hand-edited. `pnpm -F @antfu/design-storybook run build`
  regenerates and commits-worthy-updates it automatically (chained before
  `storybook build`); run `pnpm -F @antfu/design run docs:overview` directly if
  you need it without a full build. New categories must be added to the `cats`
  array in `scripts/gen-overview.mjs`.

## The components table (README + skill) is generated — don't hand-edit it

- The `## Components` table in `packages/design/README.md` and the whole
  `## <Category>` table set in
  `skills/antfu-design/references/core-components.md` live between
  `<!-- COMPONENTS:START -->` / `<!-- COMPONENTS:END -->` markers and are
  generated by `pnpm -F @antfu/design run docs:gen` (via
  `scripts/gen-components.mjs`) from the components on disk — never edit
  inside the markers by hand.
- It runs automatically before every `pnpm -F @antfu/design-storybook run build`
  and in CI's "Docs in sync" check (`nr docs:gen` + `git diff --exit-code`), so
  a stale table fails the build.
- "Documented" = has a co-located `*.stories.ts` (one table row); a `.vue` with
  no story is folded into its nearest documented sibling (e.g.
  `OverlayDropdownItem` isn't a sub-component — it has its own story — but
  `OverlayDropdownLabel` folds into `OverlayDropdown`).
- The one-line "Use it for" prose is **colocated with the component**, not in a
  separate data file that could drift on its own: a `<!-- @description ... -->`
  comment as the file's first line, right above `<script setup>` (see any
  documented component, e.g. `DisplayBadge.vue`). **Whenever you add a component
  with a story, add this comment** — the generator throws if it's missing or
  empty, so a stale/undocumented component fails the build instead of silently
  falling out of sync.
